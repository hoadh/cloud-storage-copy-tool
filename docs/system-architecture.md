# System Architecture

## Architecture Overview

The system is designed to support multiple cloud storage providers (Google Drive, OneDrive, and Dropbox) for cross-provider and intra-provider copy operations. Built on Node.js, the architecture is heavily modularized using the **Strategy Pattern** to separate the core copy orchestrator from provider-specific logic.

### Core Components

1. **CLI Layer (`src/index.js`)**
   - Entry point built using Commander.js.
   - Parses arguments and options (e.g. URI, `--dry-run`, `--verbose`).
   - Forwards parsed URIs to the system.

2. **URI Parser (`src/utils/uri-parser.js`)**
   - Handles custom URI schemas: `<provider>://<path-or-id>`.
   - Normalizes input into a `{ scheme, path }` object.

3. **Provider Factory (`src/providers/provider-factory.js`)**
   - Acts as a factory to instantiate the correct provider instance (`GoogleDriveProvider`, `OneDriveProvider`, `DropboxProvider`) based on the URI scheme.
   - Bootstraps authentication seamlessly.

4. **Copy Engine (`src/copy-engine.js`)**
   - The central orchestrator that performs directory traversal.
   - Handles identical provider scenarios by executing optimized native copy commands when supported.
   - Manages cross-provider streaming (downloading from the source stream and piping straight to the destination stream) to keep memory usage low.

5. **Provider Interface (`src/providers/base-provider.js`)**
   - Base abstract class that guarantees all providers implement standard methods: `authenticate`, `listFolderContents`, `getItemMetadata`, `downloadFileStream`, `uploadFileStream`, `createFolder`, etc.
   - Ensures any new cloud provider can be plugged in given it adheres to this interface.

6. **Authentication Managers (`src/auth/`)**
   - Decoupled modules dealing with OAuth 2.0 flows.
   - Handlers securely manage obtaining and saving tokens for their respective cloud providers.

## Execution Flow

1. User sends a command with `gdrive://source_id` and `onedrive:///destination_path`.
2. `ProviderFactory` resolves the `gdrive` and `onedrive` URIs and prepares `GoogleDriveProvider` and `OneDriveProvider` instances.
3. Both instances authenticate using cached tokens or initiate OAuth flows if tokens are missing/expired.
4. `CopyEngine` is initialized with the source and destination providers.
5. `CopyEngine` begins deep traversal of the source. For each item:
   - If folder: Recurse into it and create the corresponding folder in the destination.
   - If file: Check idempotency (skip if it exists based on criteria), and either copy natively (if same provider) or stream copy across providers.
6. Execution completes with summary logging generated by `utils/logger.js`.
